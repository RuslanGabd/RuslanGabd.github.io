How would you investigate a failed API request?

First, I would try to reproduce the issue using Postman to understand the problem. Then I would check the request details, including URL, headers, authentication, and request body. After that, I would analyze the response — especially the status code and error message. If the issue is not clear, I would check application logs and verify the status of related services. Finally, if needed, I would compare the request with a working example or escalate the issue with detailed information.

What would you do if API works in Postman but not in production?

If the API works in Postman but not in production, I would first compare the requests to identify any differences. I would check authentication, such as tokens or API keys, because they may be missing, expired, or invalid in production. Then I would verify environment-specific configurations like base URL, headers, and parameters. After that, I would check logs on the server side to identify any errors and confirm whether the request is reaching the service. I would also check network issues, such as firewall restrictions or connectivity problems. Finally, I would try to reproduce the issue and compare it with a successful request to find the root cause.
